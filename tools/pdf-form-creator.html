ℹ️ About This Tool

The ApneSoftware PDF Form Creator turns an ordinary, static PDF into a genuine fillable form — an interactive document, technically called an AcroForm, whose fields can be typed into, ticked and selected directly on screen in Adobe Reader, Chrome, Edge or any standard PDF viewer. Instead of printing a document, filling it by hand and scanning it back, you add real form fields on top of the page here, and everyone who receives the file can complete it digitally. This is exactly the capability that expensive desktop software such as Adobe Acrobat Pro, Foxit and Nitro provides, delivered as a fast, free web page that never uploads your file anywhere.

When you open a PDF it renders page by page using pdf.js, the same engine behind Firefox’s built‑in viewer, giving you an accurate canvas to design on. Building the form is intuitive: choose a field type from the palette and drag on the page to draw it exactly where it belongs, then move it, resize it from the corner handle, or duplicate it. The tool supports the field types real forms need — single‑line text for names and short answers, multiline text for addresses and comments, dropdown lists for predefined choices, checkboxes for yes/no options, radio groups for pick‑one questions, date fields for deadlines and birthdays, and signature fields for approvals. Each is created as a proper interactive widget in the exported PDF, not a drawing.

Every field is fully configurable through a dedicated properties panel. You give each one a name — the identifier that appears when the form is filled and when its data is collected — and you can add a tooltip that shows help text on hover, a default value that pre‑fills the field, and a font size and alignment for how the entered text looks. Text fields can enforce a maximum length; dropdowns and radio groups take a comma‑separated list of options. Two switches control behaviour that matters on real forms: Required marks a field that must be completed before the form can be submitted, and Read‑only locks a pre‑filled value the recipient cannot change. Fields are automatically given sensible unique names, and duplicates are de‑duplicated at save time so the resulting PDF is always valid.

To make form building fast, the tool includes ready‑made templates. A single click drops a complete registration, invoice or survey layout onto the current page, which you can then reposition and rename to suit your document — a huge head start compared with placing every field by hand. And because good forms are often reused and refined, you can export your entire field layout as JSON and re‑import it later to recreate the same design on another PDF, or export a CSV list of the fields for documentation and review. This makes the tool practical not just for a one‑off form but for a repeatable, standardised form‑production workflow.

Designing longer or multi‑page forms is comfortable. A live thumbnail strip lets you jump between pages and shows a badge with the number of fields on each one, so you can see your whole form at a glance; page arrows, a go‑to‑page box and zoom controls help you place fields precisely over existing lines and labels. A full undo and redo history means you can experiment without fear, and a statistics strip keeps a running count of the total fields and how many are marked required. When you are finished, one click writes all of your fields — with their names, defaults, options, validation and flags — into the PDF using pdf‑lib and downloads a genuine fillable form that opens correctly everywhere, including in our companion PDF Form Filler tool.

It helps to understand what makes a form field “real”. A genuine fillable PDF stores each control as a structured widget with a type, a unique name, an appearance and a set of flags, all held in the document’s AcroForm dictionary. That is very different from simply drawing a box or a line on the page: a drawn rectangle looks like a field but cannot be clicked or typed into, and any text you place is fixed forever. Because this tool creates true widgets with pdf‑lib, the fields it produces behave exactly as users expect — the cursor lands inside a text box, a dropdown opens its list, a checkbox toggles, and the entered data can later be extracted or collected programmatically. It also assigns every field a clean, unique name and preserves the order in which you add them, which becomes the natural tab order a person follows when pressing Tab to move from one field to the next. These details are what separate a professional, submission‑ready form from a document that merely looks like one, and the tool handles them for you automatically so you can focus on layout rather than PDF internals.

Above all, the tool is private by design. Every step — rendering the PDF, placing the fields and generating the AcroForm — happens entirely on your own device using the open‑source pdf.js and pdf‑lib libraries. Nothing is uploaded to a server, nothing is stored, and no account or email is required; once the page has loaded you can even work offline. That matters because the documents people turn into forms are frequently sensitive — contracts, HR and onboarding paperwork, medical and insurance intake sheets, financial applications — and they should never be sent to a third‑party server merely to have fields added. Whether you are a small business creating an order form, an HR team preparing onboarding documents, a school building a registration sheet, or anyone who wants a professional fillable PDF without costly software, this tool gives you complete control over form design with nothing to install and no privacy compromise.

❓ Frequently Asked Questions

🔒 Privacy

This tool builds your form entirely inside your browser using the pdf.js and pdf‑lib libraries. Your PDF is never uploaded to any server, never stored, and never transmitted anywhere. No account, email or signup is required, and once the page has loaded you can even work offline. Because the documents people turn into forms — contracts, HR, medical and financial paperwork — are often confidential, this local‑only design keeps them under your sole control from upload to download.

🌐 Browser Compatibility

Works in all modern browsers — Google Chrome, Mozilla Firefox, Microsoft Edge, Safari, Brave and Opera — on Windows, macOS, Linux, Android and iOS. It relies only on JavaScript and the standard File, Blob and Canvas APIs plus pdf.js. Placing and resizing fields works with both mouse and touch, so you can build forms on a phone or tablet as well as a desktop. The exported fillable PDF opens in every standards‑compliant PDF reader.

🧷 Supported PDF Standards

The tool generates standard PDF AcroForm fields — the interactive form technology defined in the PDF specification and supported by Adobe Reader, Chrome, Edge, Preview, Foxit and virtually every PDF viewer. Fields are created as proper form widgets with names, values, default values, options and flags (required, read‑only), so the output behaves as a true fillable form rather than a flattened image. Filled data can later be read back or collected by any AcroForm‑aware application.

🧩 Supported Form Field Types

Single‑line text, multiline text, dropdown (combo box), checkbox, radio button group, date (a text field configured for dates) and a signature field. Each supports a name and tooltip; text‑based fields also support default value, font size, alignment and maximum length; dropdown and radio fields take a list of options; and every field can be marked required or read‑only.

📄 Supported File Formats

Input: a standard PDF file (.pdf), including multi‑page documents. Output: a fillable AcroForm PDF with your fields embedded, plus optional exports of the field layout as JSON (re‑importable as a template) and a field list as CSV. The visible content of your original PDF is preserved — the form fields are simply added on top.
