ℹ️ About This Tool

The ApneSoftware Protect PDF tool lets you secure a PDF with a password and strong encryption, and precisely control what other people are allowed to do with it — all from your web browser, with the file never leaving your device. PDF security is one of the most important yet misunderstood parts of document handling: a contract, a bank statement, a salary slip, a medical report or a legal filing often needs to be shared, but only with the right people and only for the right purposes. This tool gives you the same professional protection found in Adobe Acrobat, Foxit and paid services such as Smallpdf and iLovePDF, using the trusted open‑source QPDF engine compiled to WebAssembly so that all encryption happens locally.

PDF protection works through two kinds of password. An open password (also called a user password) is required simply to open and view the document — without it, the file cannot be read at all. An owner password (also called a permissions password) controls what a person who has opened the file may then do, and is what you would use later to change or remove the protection. You can set either or both. A common professional pattern is to leave the open password blank so anyone can view the document, but set an owner password with restrictions so that, for example, the file can be read but not printed or copied. This tool supports every combination, with a live strength meter, a password policy checklist and a one‑click generator to help you choose a strong password, plus a confirm field so a typo never locks you out of your own file.

Encryption strength is fully in your control. AES‑256 is the default and the right choice for almost everyone — it is the same standard used to protect classified information and online banking, and it is effectively impossible to break by guessing. For compatibility with much older PDF readers you can drop to AES‑128, or to legacy RC4‑128 when a very old system requires it. Whichever you pick, the document is genuinely encrypted at the binary level, not merely flagged as protected, so its contents are unreadable to anyone without the password.

Beyond the open password, the tool exposes the PDF permission model in full. You can allow high‑quality printing, restrict recipients to low‑resolution printing only, or forbid printing entirely; you can allow or block copying of text and images; and you can choose an editing level that ranges from all changes allowed down through form filling and signing only, commenting only, page assembly only, or no changes at all. A dedicated accessibility switch lets you keep the document readable by screen readers even when other copying is blocked — an important, responsible default that many tools ignore. Before you protect, a plain‑language security summary spells out exactly what your chosen settings will and will not allow, so there are no surprises.

Sometimes you need to go the other way. If you have a PDF that you own or are authorised to unlock, and you know its password, the tool’s Remove password mode decrypts it and produces a clean, unrestricted copy. This is purely for legitimate use — the tool never attempts to crack, guess or bypass a password you do not have; it simply applies the password you provide to decrypt the file, exactly as any PDF reader would. That makes it perfect for removing protection from your own archived documents once you no longer need the restrictions, or for producing an editable copy of a form you were sent and have the password to.

It is worth being clear about what each layer of protection actually achieves, because the two mechanisms are often confused. Encrypting the document with an open password is the genuinely strong protection: the bytes of the file are scrambled, and without the password the content is mathematically unreadable — no software can display it. Permissions (printing, copying, editing), on the other hand, are enforced by the PDF reader rather than by the encryption itself; a well‑behaved reader such as Adobe Reader, Chrome or Preview honours them faithfully, which is enough for the vast majority of real‑world needs — stopping a colleague from casually printing a draft, preventing a client from copying your pricing, or keeping a signed form from being altered. For maximum secrecy, combine both: set an open password so the content is encrypted and apply the permissions you want, so the document is unreadable to outsiders and controlled for those you share the password with. Choosing a strong password is the single most important decision, which is why the tool foregrounds a strength meter, a policy checklist, a secure random generator and a confirmation field — a long, unique passphrase is what turns AES‑256 from a formality into genuinely unbreakable protection.

Above all, this tool is private and secure by design. The QPDF engine runs entirely as WebAssembly inside your browser tab, so your PDF and — crucially — your passwords are never transmitted to any server, never stored, and never logged. The temporary working copy is removed from the in‑browser filesystem immediately after your file downloads, and once the page is loaded you can even disconnect from the internet and keep working. This local‑only model is essential for a security tool, because the entire point of encrypting a document is defeated if the file or its password passes through someone else’s computer first. Whether you are a business protecting invoices and contracts, an accountant securing statements, an HR team locking down salary and personal records, or an individual safeguarding your own sensitive scans, this tool gives you real, standards‑based PDF encryption with nothing to install and no privacy trade‑off.

📖 How To Use

  1. Upload your PDF. Drag it into the drop zone or click to choose. The tool shows the file details and flags if it is already encrypted.
  2. Set passwords. Enter an open password (optional) and an owner password (required). Use the meter, policy and generator for a strong one, and confirm it.

❓ Frequently Asked Questions

🔒 Privacy & Security

This tool encrypts your PDF entirely inside your browser using the QPDF WebAssembly engine. Your file and your passwords are never uploaded to any server, never stored, and never logged. No account or signup is required, and once the page has loaded you can work offline. The temporary working copy is removed from the in‑browser filesystem right after download. Because a security tool that sent your file or password to a server would defeat its own purpose, this strictly local design is fundamental — your sensitive documents and the keys that protect them stay under your sole control at all times.

🌐 Browser Compatibility

Works in all modern browsers that support WebAssembly — Google Chrome, Mozilla Firefox, Microsoft Edge, Safari, Brave and Opera — on Windows, macOS, Linux, Android and iOS. The encryption engine loads once (a few seconds the first time) and then runs instantly. On phones and tablets the layout stacks into a single column with touch‑friendly controls, so you can protect a document from any device.

🧷 Supported Encryption Standards

AES‑256 (256‑bit Advanced Encryption Standard, PDF 2.0 / Acrobat X‑style) is the default and strongest option. AES‑128 (128‑bit, Acrobat 7+) offers wider compatibility with older readers, and RC4‑128 is available for legacy systems that predate AES support. All are implemented by the open‑source QPDF library and produce standards‑compliant encrypted PDFs that open in any conforming reader with the correct password.

📚 Supported PDF Versions

The tool reads and writes standard PDF documents from PDF 1.4 through PDF 2.0. AES‑256 encryption corresponds to the security handler introduced with PDF 2.0 / Acrobat X, while AES‑128 and RC4 correspond to earlier revisions for backward compatibility. The visible content, pages and structure of your document are preserved exactly — only the encryption and permission layer is changed.

📄 Supported File Formats

Input: a standard PDF file (.pdf). Output: the same PDF, encrypted with your chosen password(s) and permissions (Protect mode), or a decrypted, unrestricted PDF (Remove‑password mode). This tool works with the PDF format only; to secure other document types, convert them to PDF first.
