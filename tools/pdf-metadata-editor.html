ℹ️ About This Tool

The ApneSoftware PDF Metadata Editor lets you view, edit, add and remove the hidden document properties — the metadata — stored inside a PDF, all from your web browser and without uploading the file anywhere. Every PDF carries a layer of information about itself that is separate from its visible pages: its title, the author’s name, a subject line, keywords, the application that created it, the software that produced the PDF, the language, and the dates it was created and last modified. This data is what appears in a viewer’s “Document Properties” dialog, what search engines and library systems read when they index a file, and what operating systems show in file previews. Editing it properly usually requires expensive desktop software such as Adobe Acrobat Pro; this tool brings the same capability to a fast, free, private web page.

When you open a PDF, the tool immediately reads its existing metadata into a clean, two‑column editor and displays a separate, read‑only document information panel. That panel summarises the technical facts about the file — its PDF version, file size, total page count, the page size and orientation, the producer string, and the creation and modification timestamps — so you have full context before changing anything. The editable fields cover every standard Info‑dictionary property: Title, Author, Subject, Keywords, Creator and Producer, plus the document Language and both the creation and modification dates, which you can set to any value using a proper date‑and‑time picker rather than being forced to accept the current moment.

Real documents often need more than the standard fields, so the editor also supports custom metadata fields. With one click you can add your own key‑value pairs — Company, Department, Publisher, Copyright, Category, Comments or anything else your organisation uses — and rename or delete them freely. These are written directly into the PDF’s Info dictionary alongside the standard entries, so they travel with the file and are visible to any compliant PDF reader. This makes the tool genuinely useful for records management, where documents must carry consistent ownership, classification and rights information.

Privacy and anonymisation are first‑class features. A PDF’s metadata frequently reveals more than its author intended — a real name, a company username, the internal file path, or the software licence used to create it — which can be a serious problem when publishing a document publicly or submitting it to a third party. A single Remove all metadata switch strips every property and also deletes any embedded XMP metadata stream, producing a clean, anonymised file with no lingering personal information or hidden properties. You can equally use the individual fields to correct a wrong author, replace an auto‑generated title with a meaningful one, or add descriptive keywords that make the document easier to find.

To keep you in control, the editor includes a before‑and‑after comparison that lists exactly which properties you have changed, an undo/redo history, and a one‑click reset that restores the file’s original values. Built‑in validation warns you about duplicate keywords and invalid control characters so your metadata stays clean and standards‑compliant. And because metadata is often needed outside the PDF too — for a spreadsheet, an audit log or another system — you can export the current metadata as JSON, CSV, XML or TXT, and import a JSON file to apply a saved set of values as a reusable template across many documents.

Getting metadata right matters more than most people realise. For publishers and libraries, consistent Title, Author and Subject fields are what make a catalogue searchable and what appear in citation tools and reference managers. For SEO and content teams, a PDF that will be indexed by Google benefits enormously from an accurate title and descriptive keywords, because search engines fall back on this metadata when a document has no obvious on‑page heading. For legal, finance and compliance workflows, embedding Company, Copyright, Category and custom classification fields creates an auditable record of ownership and handling that travels with the file. And for everyone else, the ability to strip metadata is a simple but powerful privacy safeguard: the same document that looks anonymous on screen can quietly carry your full name, your employer, your operating‑system username and the exact software build you used, all of which a recipient can read in seconds. This tool puts every one of those scenarios within reach without a subscription, a download, or a single byte leaving your device.

Above all, the tool is private by design. It uses the open‑source pdf‑lib library to read and rewrite the document entirely inside your browser tab. Nothing is uploaded to a server, nothing is stored, and no account or email is required; once the page has loaded you can even work offline. That matters a great deal for metadata specifically, because the very information you are editing — names, organisations, internal identifiers — is exactly the kind of personal or business data that should never be sent to a stranger’s server just to be corrected or removed. Whether you are a publisher standardising document properties across a catalogue, a business stamping ownership and copyright onto its files, an SEO specialist adding titles and keywords to PDFs that will be indexed, or an individual scrubbing personal details before sharing a document, this tool gives you complete, professional control over PDF metadata with nothing to install and no privacy compromise.

❓ Frequently Asked Questions

🔒 Privacy

This tool reads and rewrites your PDF entirely inside your browser using the pdf‑lib library. Your file is never uploaded to any server, never stored, and never transmitted anywhere. No account, email or signup is required, and once the page has loaded you can even work offline. Because metadata itself often contains personal or organisational information — author names, usernames, internal identifiers — this local‑only design keeps that data under your sole control while you edit or remove it.

🌐 Browser Compatibility

Works in all modern browsers — Google Chrome, Mozilla Firefox, Microsoft Edge, Safari, Brave and Opera — on Windows, macOS, Linux, Android and iOS. It relies only on JavaScript and the standard File and Blob APIs, which have been supported for years. On phones and tablets the two‑column editor stacks into a single column and every field triggers the correct on‑screen keyboard for comfortable editing.

🧷 Supported Metadata Standards

The editor reads and writes the classic PDF Document Information dictionary (Info) — Title, Author, Subject, Keywords, Creator, Producer, CreationDate and ModDate — plus the document’s /Lang language entry and arbitrary custom Info keys. The Remove all metadata option additionally deletes any embedded XMP (Extensible Metadata Platform) stream, which is where standards such as Dublin Core, RDF and other packet‑based metadata live, ensuring an anonymised file contains no residual descriptive data.

📄 Supported File Formats

Input: standard PDF files (.pdf), including multi‑page documents. Output: the same PDF with its metadata updated (or removed), plus optional metadata exports in JSON, CSV, XML and TXT. Importing a JSON metadata file lets you reapply a saved template. The visible pages and content of your PDF are never changed — only the document properties are edited.
